# Founder Response

Hi Thabo,

Thank you for the opportunity to build the first version of your neighbourhood marketplace.

For this sprint, I focused on delivering a functional Minimum Viable Product (MVP) that demonstrates the core borrowing experience while keeping the application stable and easy to use.

## Features Completed

The following features were completed during the sprint:

- Home page displaying available items
- Search functionality
- Category filter
- Price filter (Free/Paid)
- Item details page
- Two-step booking flow with confirmation
- Sign in page
- Responsive layout for different screen sizes

## Features Deferred

I decided to postpone the following features:

- Messaging between users
- Maps
- Ratings and reviews
- Wishlist
- Referral codes
- Offline support
- Real-time updates
- Dark mode

These features would require significantly more development time and, in my opinion, would reduce the quality of the core marketplace experience if implemented within a single sprint.

## Product Decisions

I also decided not to force users to sign in before browsing items. Although capturing user information is valuable, allowing visitors to explore the marketplace first creates a better user experience and helps build trust before asking users to register.

Rather than building many incomplete features, I prioritised delivering a working, stable product that can be demonstrated, tested and expanded in future iterations.

Thank you for the opportunity to build this MVP.
