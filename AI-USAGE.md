# AI Usage Log

## Entry 1 – Planning the Application Structure

### Prompt

Help me plan the structure of a React and TypeScript marketplace application using the provided starter project.

### AI Response

The AI suggested separating the application into pages using React Router and creating reusable components for the marketplace.

### What I Changed

I simplified the suggested structure to match the project's scope and deadline while still keeping the code organised.

---

## Entry 2 – Building the Booking Flow

### Prompt

Help me build a simple booking flow in React with TypeScript.

### AI Response

The AI generated a multi-step booking process with confirmation.

### What I Changed

I reduced the flow to two clear steps with a confirmation page to keep the booking process simple and focused on the assessment requirements.

---

## Entry 3 – Handling Missing Data

### Prompt

How should I handle missing photos, ratings, prices and distances in my marketplace application?

### AI Response

The AI recommended displaying fallback values and placeholder content instead of assuming all data would always exist.

### What I Changed

I implemented logic that displays:
- "No ratings yet" when ratings are unavailable.
- "Distance unavailable" when the user's location is unknown.
- "FREE" when an item has no price.
- A placeholder image when no photos are available.

---

## AI Reflection

During development I reviewed all AI suggestions before using them. Some responses were more complex than necessary, so I simplified the code to better fit the project requirements and my own understanding. AI was useful for generating ideas and speeding up development, but I made the final implementation decisions myself.
